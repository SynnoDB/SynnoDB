 To populate the in-memory columns from the input Arrow tables, you MUST compose the framework helpers in `synno_rt::ingest` (already a dependency: `use synno_rt::prelude::*;`). Do NOT decode Arrow buffers yourself (no downcasting to typed arrays, no Decimal128 byte handling, no scale arithmetic — that is a frequent source of silent bugs like all aggregates coming out zero). `synno_rt::ingest` is the sanctioned extension point for generic ingest behavior: if a flat scalar column needs handling the current helpers do not cover, extend that helper once while preserving Arrow safe-cast, null-mask, scale, and range-check semantics, then call it from the builder. Each helper takes the table and column name, accepts Arrow physical representations through Arrow casts, and returns a Vec you index positionally into your struct-of-arrays. Use Arrow casts to decode correctly, then store the narrowest correct Rust representation chosen by the storage plan:
  - DECIMAL / money / quantity -> `scaled_integer::<T>(&tables.TBL, "COL", DECIMALS)?` (Vec<T> of value*10^DECIMALS, exact fixed-point; choose T as the narrowest safe integer such as i16/i32/i64, and use the column's decimal scale);
  - integer/code/key columns -> `as_integer::<T>(&tables.TBL, "COL")?` (choose T from i8/u8/.../i64/u64 based on the declared/ranged domain; do not default to i64 when the plan proves a narrower type is correct);
  - string columns  -> `as_string(&tables.TBL, "COL")?`;
  - date columns    -> `as_date_days(&tables.TBL, "COL")?` (i32 days since 1970-01-01);
  - floating columns -> `as_double(&tables.TBL, "COL")?`.
Every helper returns `Result`, and a cast that cannot be done - or a value that overflows the target - is an Err rather than a wrong number, so propagate with `?`. For a NULLABLE column use the `*_nullable` variant, which also returns the validity mask. Declare each Database field with the matching narrow element type.
